```

║   正确流程：                                                  ║
║   1. Fork 主仓库到你的账户                                    ║

```

### 提交格式

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   Commit Message 格式                                       │
│   ─────────────────────                                    │
│                                                             │
│   类型(范围): 简短描述                                      │
│                                                             │
│   类型：                                                    │
│   - feat: 新功能/新内容                                     │
│   - fix: 修复错误                                           │
│   - docs: 文档更新                                          │
│   - style: 格式调整（不影响内容）                           │
│   - refactor: 重构                                          │
│   - restore: 遗迹修复                                       │
│                                                             │
│   范围：                                                    │
│   - spatial: 空间架构                                       │
│   - faction: 派系内容                                       │
│   - ops: 行动部门                                           │
│   - reception: 接待部门                                     │
│   - tourism: 观光区                                         │
│   - commerce: 商业中心                                      │
│   - academy: 教育学院                                       │
│                                                             │
│   示例：                                                    │
│   feat(spatial): 添加星云花园夜间描述                       │
│   fix(faction): 修正ECHO联合体时间线错误                    │
│   restore(tourism): 修复旧核心遗址展示内容                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Git提交规范

```
╔═══════════════════════════════════════════════════════════════╗

║                                                               ║
║   正确流程：                                                  ║
║   1. Fork 主仓库到你的账户                                    ║
║   2. 在你的 Fork 中进行修改                                   ║
║   3. 完成后，先在你的仓库中展示                               ║
║   4. 通知维护团队审核                                         ║
║   5. 审核通过后，由维护团队同步到主仓库                       ║

### 提交格式

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   Commit Message 格式                                       │
│   ─────────────────────                                    │
│                                                             │
│   类型(范围): 简短描述                                      │
│                                                             │
│   类型：                                                    │
│   - feat: 新功能/新内容                                     │
│   - fix: 修复错误                                           │
│   - docs: 文档更新                                          │
│   - style: 格式调整（不影响内容）                           │
│   - refactor: 重构                                          │
│   - restore: 遗迹修复                                       │
│                                                             │
│   范围：                                                    │
│   - spatial: 空间架构                                       │
│   - faction: 派系内容                                       │
│   - ops: 行动部门                                           │
│   - reception: 接待部门                                     │
│   - tourism: 观光区                                         │
│   - commerce: 商业中心                                      │
│   - academy: 教育学院                                       │
│                                                             │
│   示例：                                                    │
│   feat(spatial): 添加星云花园夜间描述                       │
│   fix(faction): 修正ECHO联合体时间线错误                    │
│   restore(tourism): 修复旧核心遗址展示内容                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
