## 第一章：Git 提交规范

### Commit Message 格式

```
类型(范围): 简短描述

详细描述（可选）

相关Issue（可选）
```

### 类型 (Type)

| 类型 | 说明 | 示例 |
|------|------|------|
| `feat` | 新功能/新内容 | 添加新空间、新角色 |
| `fix` | 修复错误 | 修正错别字、逻辑错误 |
| `docs` | 文档更新 | README更新、注释添加 |
| `style` | 格式调整 | 排版、标点（不影响内容） |
| `refactor` | 重构 | 结构调整、文件重组 |
| `restore` | 遗迹修复 | 恢复损坏的档案 |
| `art` | 艺术创作 | ASCII艺术、视觉设计 |

### 范围 (Scope)

| 范围 | 说明 |
|------|------|
| `spatial` | 空间架构 |
| `faction` | 派系内容 |
| `ops` | 行动部门 |
| `reception` | 接待部门 |
| `tourism` | 观光区 |
| `commerce` | 商业中心 |
| `academy` | 教育学院 |
| `culture` | 文化区 |
| `port` | 港务区 |
| `lodging` | 住宿区 |

### 示例

```
feat(spatial): 添加星云花园夜间描述

为星云花园添加了夜间特别场的详细描述，
包括发光植物和夜间活动安排。

Related: #42
```

```
fix(faction): 修正ECHO联合体时间线错误

2049年事件日期从3月改为4月，
与其他档案保持一致。
```

```
restore(tourism): 修复旧核心遗址展示内容

恢复了崩溃前的控制室布局描述，
基于2048年的备份数据。
```

## 第三章：遗迹修复项目

### 什么是遗迹修复？

遗迹修复是指恢复"大崩溃"中损坏或丢失的数字档案。这是一种特殊的贡献形式，需要：

- 研究现有档案，理解上下文
- 根据线索推断缺失内容
- 以符合世界观的方式填补空白

### 当前修复项目

| 项目 | 优先级 | 状态 | 负责人 | 名额 |
|------|--------|------|--------|------|
| 旧核心数据恢复 | 高 | 进行中 | @DataArch | 3 |
| 黄金时代商店复原 | 中 | 招募中 | 待定 | 5 |
| 崩溃前通信记录 | 中 | 招募中 | 待定 | 2 |
| 早期AI对话日志 | 低 | 规划中 | 待定 | 2 |
| 创始者个人档案 | 低 | 规划中 | 待定 | 1 |

### 参与方式

1. 在 Issues 中查看当前项目
2. 选择感兴趣的项目，申请参与
3. 接受培训（修复方法论）
4. 领取任务包
5. 在个人Fork中完成工作
6. 提交审核
7. 获得贡献积分

### 贡献积分奖励

| 贡献类型 | 积分 |
|----------|------|
| 小型修复（<500字） | 10分 |
| 中型修复（500-2000字） | 30分 |
| 大型修复（>2000字） | 50分 |
| 重要发现 | 100分 |

积分可兑换：
- VTK代币
- 特殊访问权限
- 贡献者徽章
- 优先招聘机会
