### 问题1：文本质量参差不齐

**表现**：
- 优质文件有ASCII艺术、系统代码风格、详细内容（如疗愈区2025蓝图期）
- 简陋文件只有几行干巴巴的文字（如教育区2025蓝图期）

**对比示例**：

| 优质 | 简陋 |
|------|------|
| 有ASCII艺术头部 | 没有 |
| 有[SYSTEM_NOTE]等标签 | 没有 |
| 详细的背景故事 | 只有列表 |
| 1000+行 | <30行 |

**原因**：
- 没有统一的写作模板
- 不同时期写的风格不一致

## 📋 解决方案

### 方案1：建立写作规范

创建一个"写作指南"文件，包含：
1. 文件头部模板（ASCII艺术）
2. 标签系统（[SYSTEM]、[HELPER]等）
3. 段落结构规范
4. 命名规范

### 方案2：创建Prompt模板

存放位置：`_系统文档/写作模板/`

内容：
- 空间档案模板
- 三时期档案模板
- 派系档案模板
- 杂志文章模板

### 方案3：优先级排序

| 优先级 | 任务 | 预计工作量 |

## 📝 写作风格参考

### 推荐风格（以疗愈区为标杆）

```markdown
# [ARCHIVE_LOG: 区域_时期_主题]
# ACCESS_LEVEL: PUBLIC // STATUS: 状态

---
## // PROJECT_CODE: 代号 //
### 标题 (年份)

> **[SYSTEM_NOTE]**: 简介说明...

---

### // 章节标题 //

**[关键词]**: 内容...

---
```

### 必要元素
1. ASCII艺术或系统风格头部
2. [SYSTEM_NOTE]引言
3. 表格数据
4. 引用块（故事/证言）
5. 统一的分隔线
